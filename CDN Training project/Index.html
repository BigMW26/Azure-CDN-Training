7)     Was asset served by Microsoft CDN, WGET install and Impression Log Search

a.      Install WGET by downloading the complete setup package

                                                    i.     http://gnuwin32.sourceforge.net/packages/wget.htm

b.     Use WGET to get the ref string response header

                                                    i.     Open CMD prompt and change directory to C:\Program Files (x86)\GnuWin32\bin

       cd C:\Program Files (x86)\GnuWin32\bin

                                                  ii.     Run WGET cmd

       Sample command: wget -S -O NUL no-check-certificate https://<alias>cdn.azureedge.net/index_files/image001.png

c.      Follow instruction for use of Impressions Log Search tool

                                                    i.     [HowTo] Check if Asset was served from Microsoft CDN Cache  (Web view)

                                                  ii.     Impression Log Search: https://www.afdcp.com/ImpressionLogSearch

8)     Testing different regions

a.      Install WGET on one of your VMs

b.     Follow the WGET process with Impression Log Search to see if the POP thats serving the asset has cached the image as well.

                                                    i.     Try a different image file following the folder structure of the project files.

c.      Use WhatsMyDNS to determine the IP addresses of your CDN endpoint in different regions

                                                    i.     https://www.whatsmydns.net

                                                  ii.     Run WGET cmd to test from the secondary region

       Sample cmd: wget -S -O NUL -header=Host:<alias>cdn.azureedge.net http://<ipaddress>/index_file/image001.png

d.     Use Dig to get additional data like CNAME and other DNS records associated with the CDN URL. Helpful in cases with custom DNS configured

                                                    i.     http://digdns.com

                                                  ii.     OneNote Link for instruction: Common External Tools  (Web view)

9)     Using Fiddler for captures

a.      Install Fiddler on your local machine or VM you want to test from: (Need Fiddler 4.4.9.2 or higher)

                                                    i.     https://www.telerik.com/fiddler

b.     Install FDTrace

                                                    i.     \\tkfiltoolbox\tools\FDTraceInstaller\1.0.10

       If on local machine - Go to directory, right click the file and Run as Administrator

       If on VM Go to directory > Copy file to VM > Run as Administrator (The Impression log search must be used to examine log if performed on VM as this request doesnt originate on corp network)

11)  Common Scenarios and Escalations

a.      Portal/Setup issues

b.     Cache/Expiration and purging issues

c.      Service not Available Our services aren't available right now  (Web view): usually a matter of waiting after creating a new CDN profile and endpoint.

                                                    i.     Will require ICM if correctly configured and unavailable after 12 hours.

d.     Verizon Cache settings: Verizon Cache Settings and Expiration  (Web view)

                                                    i.     This may require escalation to Verizon

e.      Upgrading profile from Standard to Premium: https://docs.microsoft.com/en-us/azure/cdn/cdn-migrate

                                                    i.     This requires escalation to MS PG: https://aka.ms/ALLCDN_ICM (to be added to ASC in the near future)

f.       Edge node IP whitelisting on Origin server

                                                    i.     Azure CDN Edge Server IP ranges  (Web view)

                                                  ii.     Edge Nodes List: https://docs.microsoft.com/en-us/rest/api/cdn/edgenodes/list

g.     SSL Enablement Stuck for Akamai

                                                    i.     Run Kusto query in OneNote: SSL enablement stuck  (Web view)

       If the substate shows PendingDomainControlValidationRequestApproval, that means Akamai failed to validate the custom domain.

       May require an ICM to PG for redirection path as explained in OneNote if CNAME record is correct or corrected and still cannot validate custom domain
